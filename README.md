# 纽斯光谱可视化工具

一个面向 Noo-Psyche 珊瑚灯配置的前端可视化编辑工具。  
它可以编辑 24 组时点光谱数据、生成二维码、从二维码或设备报文回填配置，并提供“模拟日出日落”的时间预设能力。

## 主要功能

- 编辑 24 组时点数据
  - 固定 24 个小时点
  - 每个时点可设置分钟
  - 六通道亮度支持滑块和十进制输入
- 预设方案
  - 内置默认示例
  - 内置 `SPS` 和 `LPS` 方案
- 数据趋势图
  - 展示六个光通道在 24 个时点上的变化趋势
  - 图表高度会跟随页面布局动态重绘
- 二维码能力
  - 根据当前配置实时生成二维码
  - 支持保存二维码 PNG
  - 支持从二维码图片导入配置
- 原始串与设备报文
  - 支持从原始串回填表格
  - 支持从设备返回报文中自动提取 24 组配置
  - 支持复制当前原始串
- 模拟日出日落
  - 可直接填写开灯时间和关灯时间
  - 可按经纬度和时区偏移计算时间
  - 支持把常用配置保存为本地预设

## 数据说明

### 1. 原始串格式

原始串格式如下：

```text
手动模式六通道亮度头(12位HEX)#24组16位时序数据
```

例如：

```text
323232323232#00000000000000000100000000000000...
```

其中：

- `323232323232` 是手动模式下六通道亮度头
- 这 12 位头部会在界面中以十进制输入，导出时自动转换为两位 HEX
- `#` 后面是 24 组时序数据，每组固定 16 位

### 2. 单组 16 位数据结构

每组 16 位数据按 8 个字节组成：

```text
小时 分钟 白光 深蓝光 绿色光 UV 浅蓝光 红色光
```

说明：

- 小时固定为 `00` 到 `17` 的 24 个完整时点
- 分钟可调
- 六个光通道范围为 `0-100`

### 3. 设备报文导入

工具支持直接粘贴设备返回报文，例如：

```text
abaa1008......bb
```

当前实现会自动：

- 识别 `abaa` 起始标记
- 提取手动模式亮度头
- 读取分组数
- 截取 24 组时序数据
- 忽略后续设备标识尾部

如果输入的是完整设备报文，无需手动裁剪。

## 模拟日出日落

“模拟日出日落”区域用于快速调整当前方案的开灯和关灯节奏。

支持两种方式：

1. 直接手填开灯时间和关灯时间
2. 输入纬度、经度和时区偏移后自动计算

经纬度输入支持：

- 十进制格式
  - 例如 `23.1291`
  - 例如 `113.2644`
- 度分秒格式
  - 例如 `23°07'45"N`
  - 例如 `113°15'52"E`

预设能力：

- 可把常用地点或时间方案保存为本地预设
- 预设保存在浏览器本地 `localStorage`
- 选择预设后会自动载入

注意：

- 该功能只调整当前方案的时间映射
- 24 个小时点结构会保持完整，避免回传设备时破坏协议格式

## 界面说明

- `24 组时间点编辑`
  - 主编辑区
- `数据趋势图`
  - 查看六通道变化趋势
- `二维码预览`
  - 根据当前配置生成二维码
- `操作区`
  - 预设、导入、导出、显隐控制
- `模拟日出日落`
  - 时间预设与地点预设
- `汇总 / 原始串`
  - 默认隐藏，可按需展开

## 本地开发

安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```

构建生产版本：

```bash
npm run build
```

本地预览构建结果：

```bash
npm run preview
```

## 项目结构

```text
.
├─ index.html
├─ package.json
├─ vite.config.js
├─ src/
│  ├─ main.js
│  └─ styles.css
└─ dist/
```

说明：

- 主要逻辑在 `src/main.js`
- 主要样式在 `src/styles.css`
- `dist/` 为构建产物

## 当前限制

- 目前是单页前端工具，没有后端
- 常用预设只保存在当前浏览器，不会跨设备同步
- 日出日落计算依赖输入的经纬度和时区偏移，本身不连接在线天文服务
